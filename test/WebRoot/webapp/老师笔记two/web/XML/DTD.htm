什么是DTD

  • DTD是Document Type Definition，文档类型定义
  • DTD用了描述XML文档的结构

DTD有包含如下结构：

  • 元素(ELEMENT)的定义规则
  • 元素之间的关系规则
  • 属性(ATTLIST)的定义规则
  • 可使用的实体(ENTITY)或符号(NOTATION)规则

为什么要使用DTD

  • 有了DTD，每个XML文件可以携带一个自身格式的描述。
  • 有了DTD，不同组织的人可以使用一个通用DTD用来交换数据
  • 应用程序可以使用一个标准DTD校验从外部世界接收来的XML数据是否有效
  • 可以使用DTD校验自己的XML数据

DTD文档的声明以及引用

  1. 内部DTD文档：&tl;!DOCTYPE 根元素[定义内容]>
  2. 外部DTD文档：&tl;!DOCTYPE 根元素 SYSTEM "DTD文件路径">，可以是远程路径(注意SYSTEM必须大写)
  3. 内外部DTD文档结合：&tl;!DOCTYPE 根元素 SYSTEM "DTD文件路径" [定义内容]>

DTD元素与属性定义详细

元素声明语法

  • <!ELEMENT 元素名称 类别>
  • <!ELEMENT 元素名称 (元素内容)>

元素类别

  • EMPTY：空元素，声明示例：<!ELEMENT hr EMPTY>。表示该元素不能有任何的子元素或文本，但可以有属性
  • ANY：带有任何内容的元素，声明示例：<!ELEMENT note ANY>。表示该元素可以包含任何在DTD中定义的元素内容
  • #PCDATA：只有#PCDATA的元素，声明示例：<!ELEMENT a (#PCDATA)>。表示该元素可以包含任何字符数据，但不能包含子元素
  • 带有子元素（序列）的元素(纯元素类型)：只有包含子元素，声明示例：<!ELEMENT book (author,title,page)>。表示该元素只包含子元素，不能包含字符数据
  • 混合型：可以有#PCDATA，又可以有子元素的元素，声明示例：<!ELEMENT book (#PCDATA|author|title)>。

属性声明语法

  • <!ATTLIST 元素名称 属性名称 属性类型 默认值>
  • 属性声明是不会单独出现的，它必须依赖于元素

#PCDATA、CDATA、CDATA节的区别

  • CDATA节：用于把整段文本解释为纯字符数据而不是标记
  • CDATA：表示元素属性的类型，注意，没有#号
  • #PCDATA：表示元素的类型，注意，有#号

NMTOKEN和NMTOKENS

  • NMTOKEN：是CDATA的一个子集，表示属性值必须是英文字母、数字、句号、破折号、下划线或冒号，属性值不能含有空格。
  • NMTOKENS：与NMTOKEN类似，包含多个由空格分隔的字符

实体

  • NMTOKEN：是CDATA的一个子集，表示属性值必须是英文字母、数字、句号、破折号、下划线或冒号，属性值不能含有空格。
  • NMTOKENS：与NMTOKEN类似，包含多个由空格分隔的字符

参考代码

<!-- dtd_test.dtd文件 -->
<?xml version="1.0" encoding="UTF-8"?>
<!--
DTD文件的元素、属性详解
-->

<!-- 定义元素用ELEMENT，注意必须大写 ，元素与其子元素的定义顺序可以不定-->
<!-- 根元素books中可以有0到多个book元素，如果没有后面的*号，那么该根元素必须且只能有一个book元素 -->
<!ELEMENT books (book*)>
<!-- ELEMENT 元素名  小括号表示其子元素的集合，每个子元素之间用逗号隔开表示每个子元素都必须存在，而且顺序也不可改变 -->
<!ELEMENT book (author,title,page)>
<!ELEMENT author (#PCDATA)>
<!ELEMENT title (#PCDATA)>
<!ELEMENT page (#PCDATA)>

<!-- dtd_test.xml文件 -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE books SYSTEM "dtd_test.dtd">

<books>
	<book>
		<author></author>
		<title></title>
		<page></page>
	</book>
</books>
