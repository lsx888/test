JavaSE-简介

最后修改时间：2016-05-13 14:40:31

JAVA是由sun公司推出的(2010年被oracle公司以74亿美元收购)


JAVA主要分为：

  • JAVA SE: Java Standard Edition -- Java基础版本
  • JAVA EE: Java Enterprise Edition -- Java企业版本
  • JAVA ME: Java Micro Edition -- Java微观版

JDK：

  • JDK: java Development Kit(java开发包)
  • JRE: java Runtime Environment(java运行环境)
  • JDK包含了JRE

JDK版本：

  • JDK 1.4
  • JDK 1.5(5.0)--别名Tiger,老虎
  • JDK 1.6(6.0)--别名Mustang,野马
  • JDK 7.0 -- 别名 海豚
  • JDK 8.0
  • JDK 9.0 计划2016年发布

JAVA配置：

  • 下载JDK http://www.oracle.com/
  • 安装JDK
  • 配置环境变量(可以是用户变量，也可以是系统变量)指向JDK安装目录的bin目录
    • JAVA_HOME：JDK根目录;如：C:\Program Files\Java\jdk1.7.0_79
    • path：JDK根目录/bin;
  • 通过运行，输入cmd打开命令行窗口，输入java -version显示出Java版本信息

编写JAVA程序：

  • 可以直接使用windows记事本来编写Java程序，也可以使用Editplus，UltraEdit、notepad++等高级文本编辑工具编写Java程序.
  • 还可以使用专业的IDE(Integrated Development Environment)编写，如eclipse、myeclipse
  • 所有的Java代码，其后缀都是以java结尾

Java程序的执行过程分为两步：

  • 编译：javac test.java(执行后生成test.class文件为java的执行文件)
    • Class文件是字节码文件，程序最终执行的就是这个字节码(bytecode)文件
  • 执行：java test(注意这里没有.java)

JAVA跨平台：

  • Java是跨平台的语言，真正执行的不是二进制代码，而是字节码
  • Java是跨平台的，而JVM不是跨平台的(JVM是由C语言编写的)
  • Java之所以能够做到跨平台，本质原因在于JVM不是跨平台的

JAVA编程规范：

  • 所有的文件后缀均为.java
  • 文件名与类名必须相同
  • 类名首字母要大写
  • 最好一个文件一个类

JAVA的编辑器：(轻量级)

  • Windows: notepad, editplus, ultraedit, gvim
  • Linux: vi, vim, gedit

一些常见的英文缩写

  • JAVA SE: Java Standard Edition -- Java基础版本
  • JAVA EE: Java Enterprise Edition -- Java企业版本
  • JAVA ME: Java Micro Edition -- Java微观版
  • JVM：Java Virtual Machine -- Java虚拟机
  • JDK: java Development Kit -- java开发包
  • JRE: java Runtime Environment -- java运行环境
  • IDE: Integrated Development Environment -- 集成开发环境
  • JNDI: Java Naming and Directory Interface -- Java命名和目录接口
  • AOP: Aspect-Oriented Programming -- 面向切面(方面)编程
  • OOP: Object Oriented Programming -- 面向对象编程
  • OOD: Object Oriented Design -- 面向对象设计
  • POJO：Plain Old Java Object -- 简单的Java对象
